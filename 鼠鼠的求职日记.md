看了raft 优化的部分

* 读优化
  * read index 读的时候记录 commit id 然后马上发一个心跳，回复了就等 apply 到 read index 的时候读。
  * lease read 可以认为再 start + election timeout 期间自己都是 leader，可以直接读。
  * follower read 配合 read index，需要向 leader 询问 commit id。
* 投票优化
  * prevote 就是正式投票前预投一次。网络分区的时候就不会 term 一直增长了

dp 要注意的事情：

* 初始化
* 边界判断
* 循环顺序

链表相关

* 删除重复链表，只留一个
  * 下一个是不是和现在的一样
    * 一样 cur->next = cur->next->next
    * 不一样，cur 可以移动了，cur = cur->next
* 删除重复链表
  * cur 指向上一个合法的节点或者 dummy
  * 下一个和下下一个是不是一样
    * 一样，一直找到第一个不一样的 cur->next = cur->next->next
    * 然后统一 cur = cur->next
